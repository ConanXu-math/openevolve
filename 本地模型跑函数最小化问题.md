# OpenEvolve 本地模型 (Ollama) 实战运行指南

本文档记录了如何使用本地部署的 Ollama 模型（qwen2.5:7b）运行 OpenEvolve 进化框架，完成函数极小值搜索任务的全流程。

## 1. 环境准备 (Prerequisites)

### 1.1 基础软件
- **Python**: 3.10+
- **OpenEvolve**: `pip install openevolve`
- **Ollama**: 需已安装并启动服务

### 1.2 模型准备
确保本地已下载目标模型（这里使用 qwen2.5:7b，平衡了速度与能力）：
```bash
ollama pull qwen2.5:7b
```
验证模型是否可用：
```bash
ollama list
# 应显示 qwen2.5:7b
```

## 2. 代码与配置修改 (Configuration)

为了适配本地环境，我们需要对源码和配置文件进行关键修改。

### 2.1 修改入口脚本 (`openevolve-run.py`)
为了防止本地 `localhost` 请求被系统代理拦截，需要在脚本最上方添加环境变量设置：

```python
import os
# 避免本地 Ollama 请求被代理拦截
os.environ.setdefault("NO_PROXY", "localhost,127.0.0.1")

import sys
from openevolve.cli import main
# ...
```

### 2.2 修改任务配置 (`examples/function_minimization/config.yaml`)

需要做三处关键调整：**切换模型**、**增加超时时间**、**关闭 Diff 模式**。

```yaml
# 1. LLM 配置：指向本地 Ollama
llm:
  primary_model: "qwen2.5:7b"        # 主模型
  primary_model_weight: 1.0
  secondary_model: "qwen2.5:7b"      # 辅模型 (可选，设为0权重即不启用)
  secondary_model_weight: 0.0
  api_base: "http://localhost:11434/v1" # Ollama 接口地址
  api_key: "ollama"                  # 占位符，不能为空
  temperature: 0.7
  max_tokens: 16000
  timeout: 300                       # [重要] 增加超时时间至 300秒，防止本地推理慢导致断连

# 2. 进化设置：关闭 Diff 模式
# 本地小模型写 Diff 格式容易出错，建议使用全量重写模式
diff_based_evolution: false          # 改为 false
max_code_length: 20000
```

## 3. 运行进化 (Execution)

在终端执行以下命令启动进化过程：

```bash
# 设置 API KEY 环境变量 (Ollama 不需要真实 Key，但程序检查需要有值)
export OPENAI_API_KEY="ollama"

# 运行命令
python openevolve-run.py \
  examples/function_minimization/initial_program.py \
  examples/function_minimization/evaluator.py \
  --config examples/function_minimization/config.yaml \
  --iterations 10
```

## 4. 常见问题排查 (Troubleshooting)

### Q1: 报错 "No valid diffs found in response"
*   **原因**: 模型没能按照 Diff 格式（`--- original`, `+++ modified`）输出代码，或者输出了乱码。
*   **解决**: 在 `config.yaml` 中设置 `diff_based_evolution: false`。

### Q2: 报错 "LLM generation failed" 或 "Timeout"
*   **原因**: 本地显卡/CPU 推理速度较慢，超过了默认的 120秒 限制。
*   **解决**: 在 `config.yaml` 的 `llm` 部分，将 `timeout` 增加到 `300` 或更高。

### Q3: 报错 "Connection refused"
*   **原因**: Ollama 服务未启动，或被代理拦截。
*   **解决**: 
    1. 确保后台运行了 Ollama。
    2. 确保 `openevolve-run.py` 中设置了 `NO_PROXY`。

## 5. 结果验证 (Results)

*   **详细指标**: `best/best_program_info.json`
    *   查看 `combined_score` 和 `value_score`。
